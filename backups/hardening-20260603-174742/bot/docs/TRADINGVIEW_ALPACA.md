# TradingView + Alpaca Paper Integration

Educational and paper-trading use only. This bot is configured to reject live Alpaca endpoints by default.

## What Was Added

- `bot/core/velez_strategy.py`: Oliver Velez institutional candle-play engine.
- `bot/webhook_server.py`: FastAPI webhook receiver for TradingView alerts.
- `bot/brokers/alpaca.py`: Alpaca paper REST adapter for account checks and order submission.
- `bot/tradingview/velez_core_alerts.pine`: Pine Script alert source.
- `bot/.env.example`: required environment variables.

## Guardrails

- Orders are proposed only unless both are true:
  - `webhook.execute_orders: true` in `bot/config.yaml` or `--execute` is passed.
  - `VELEZ_EXECUTE_ORDERS=true` is present in the environment.
- Alpaca base URL must remain `https://paper-api.alpaca.markets` unless `paper_only` is intentionally disabled.
- Webhook auth is required through `VELEZ_WEBHOOK_SECRET`.
- Position sizing uses the smaller of account-equity percentage risk and `max_dollar_risk_per_trade`.
- Stops must be valid and inside `risk.max_stop_pct`.
- TradingView retries are deduplicated by alert id or payload hash.

## Run The Webhook Locally

```bash
export APCA_API_KEY_ID="your-paper-key"
export APCA_API_SECRET_KEY="your-paper-secret"
export VELEZ_WEBHOOK_SECRET="long-random-token"
export VELEZ_EXECUTE_ORDERS=false

python3 -m bot.main --config bot/config.yaml webhook --host 127.0.0.1 --port 8080
```

Health check:

```bash
curl -s http://127.0.0.1:8080/health
```

When ready to submit paper orders:

```bash
export VELEZ_EXECUTE_ORDERS=true
python3 -m bot.main --config bot/config.yaml webhook --host 127.0.0.1 --port 8080 --execute
```

## TradingView Connection Options

### Option 1: Pine Script Computes Signals, Bot Executes

Use `bot/tradingview/velez_core_alerts.pine` in TradingView. Create an alert with:

- Condition: `Velez Core Institutional Alerts`, then `Any alert() function call`.
- Webhook URL: `https://YOUR_PUBLIC_URL/webhook/tradingview/YOUR_SECRET_TOKEN`.
- Message: leave as generated by the script alert.

This is the cleanest TradingView path because TradingView calculates the SMA/tail/180/Elephant logic plus the expanded V6.15 plays on its servers and posts JSON to the bot.

The Pine script now emits these play names:

```text
elephant_bar
bull_180
bear_180
bottoming_tail
topping_tail
velez_buy_setup
velez_sell_setup
nrb_acorn
color_change_add
fab4_trap_breakout
failed_new_high
failed_new_low
opening_gap_go
opening_gap_fade
time_space_breakout
```

`color_change_add` includes add-to-winner metadata. The server will only submit it as a 50% add when an existing profitable paper position is present and the add stop shows risk mitigation.

### Option 2: TradingView Sends Raw Bars, Bot Computes Signals

Send this JSON from an alert message:

```json
{
  "mode": "bar",
  "symbol": "{{ticker}}",
  "timeframe": "{{interval}}",
  "timestamp": "{{time}}",
  "open": {{open}},
  "high": {{high}},
  "low": {{low}},
  "close": {{close}},
  "volume": {{volume}}
}
```

Webhook URL:

```text
https://YOUR_PUBLIC_URL/webhook/tradingview/YOUR_SECRET_TOKEN
```

This requires the webhook process to stay warm long enough to build SMA history.

### Option 3: Public Tunnel To Local Bot

Run the bot locally, expose it with a tunnel such as Cloudflare Tunnel or ngrok, and put that public HTTPS URL into TradingView. This is convenient for paper trading but should be kept token-protected.

### Option 4: Deploy The Webhook

Deploy the webhook process on a VPS, Render worker/web service, Fly.io, or another always-on host. This is more reliable than leaving a laptop running.

### Option 5: Bypass TradingView

Use Alpaca market data directly and run the strategy on incoming bars. This removes webhook latency and avoids TradingView alert limits. The current adapter already has the execution side; a streaming data loop can be added next.

### Option 6: Vision Over TradingView Charts

Leaving TradingView open for visual chart reading can help with discretionary review, screenshots, or supervision, but it should not be the primary execution trigger. Pixel/vision execution is slower and less auditable than webhooks or broker data.

## Source Notes

- TradingView webhooks send an HTTP POST to your configured webhook URL. If the alert message is valid JSON, TradingView sends it as `application/json`.
- Pine Script `alert()` supports dynamic alert messages.
- Alpaca paper orders are submitted to `/v2/orders`; OTO/bracket orders attach stop-loss protection to entry orders.
